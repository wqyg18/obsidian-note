> **如果外层有序扫描，内层是在“当前可用候选集合”里找最小/最大，那么通常可以用堆优化双重循环。** 

```txt
原来：
每次 for x，都从所有 item 里找一个最合适的

优化：
随着 x 增大，把新变可用的 item 加入堆；
把已经失效的 item 弹掉；
堆顶就是当前最合适的 item。
```

例如从: 
```python
for s in nums:
    for interval in intervals:
        找一个还没用过，并且 l <= s <= r 的区间
```
尽管可以加used数组, 但是这个只是确保不会重复使用, 但**不能减少扫描**, 所以复杂度还是`O(n*m)`

使用堆优化, 则是

```python
intervals.sort(key=lambda x: x[0])  
nums.sort()
for s in nums:  
	# 把所有 l <= s 的内容加入候选池
	# 因为nums是升序, 所以这里l <= s, 往后面会一直成立
	while idx < n and intervals[idx][0] <= s:  
		heapq.heappush(heap, intervals[idx][1])  
		idx += 1  
	  
	# 删除已经无法匹配当前 s 的内容
	# 尽管l <= s一直成立, 但是右端点可能不匹配了, 需要pop
	while heap and heap[0] < s:  
		heapq.heappop(heap)  
	  
	# 注意这里是否需要pop, 需要看能不能再次使用
	if heap:  
		heapq.heappop(heap)  
		res += 1
```
这里, 每一个intervals中的元素, 全部只会入堆, 出堆各一次, 那么复杂度就是 `O(n logn)`
