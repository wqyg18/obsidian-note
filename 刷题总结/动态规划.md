一般定义可能是:
+ 以`i`为结尾的dp对应的结果
+ 从开始到`i`的dp的对应的结果

他们的区别是, 是否结果中, 硬性包括`i`

还可能是 从`i`开始到`j`结束
一维的字符串也可能是需要二维dp来处理的

一般而言, 我习惯与初始化1-based的dp, 那么就需要注意dp`[0]`相关的初始化了
比如找路的问题, 只能往右,往下走, 此时因为多了一圈, 那么你要注意, 不要让`dp[0][j]`和`dp[i][0]`不正确的参与到结果中, 因为是要看起点是哪里, 是左上角, 还是第一行随便, 如果初始话直接为`0`, 那么可能导致直接从最后一行的左上角出发了

注意返回的应该是什么, 可能是
```python
dp[n]
dp[0]
max(dp)

# 二维
max(dp[0])
max(dp[n])
dp[m][n]
max(max(row) for row in dp)
```

还需要仔细考虑, dp的定义, 有时候可能倒着定义更加直观 ,比如 `dp[i]`代表从`i`题开始做的话, 一共可以得多少分, 这种定义就完全不用考虑之前的了, 然后再倒序遍历


## 初始化
初始化的时候, 注意, 这两种初始化对应的复杂度差的很多, 切分很重
```python
for i in range(1, len(s1) + 1):

	dp[i][0] = s1[:i] == s3[:i]

for j in range(1, len(s2) + 1):

	dp[0][j] = s2[:j] == s3[:j]
```

应该优化一下:

```python
for i in range(1, len(s1) + 1):
    dp[i][0] = dp[i - 1][0] and s1[i - 1] == s3[i - 1]

for j in range(1, len(s2) + 1):
    dp[0][j] = dp[0][j - 1] and s2[j - 1] == s3[j - 1]
```
